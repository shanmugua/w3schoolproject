Git Tutorial

Git is a version control system. Git helps you keep track of code changes. Git is used to collaborate on code.

✅ Why use Git? Track every change, collaborate with teams, revert mistakes, branch for features, and deploy with confidence.

Git Config

Before using Git, you should configure your identity. This is important because every Git commit uses this information.

Terminal
git config --global user.name "Your Name"
git config --global user.email "you@example.com"

Git Init & Clone

Terminal
# Start a new repository
git init

# Clone an existing repository
git clone https://github.com/user/repo.git

Git Add — Staging Changes

The staging area is like a preparation area where you gather changes before making a commit.

Terminal
# Add a specific file
git add index.html

# Add all changed files
git add .

# Check status
git status

Git Commit

A commit is a snapshot of your project at a specific point in time. Always write a meaningful commit message.

Terminal
# Commit staged changes
git commit -m "Add navigation menu"

# Add all + commit in one step
git commit -am "Fix button styling"

# View commit history
git log --oneline

Git Branch

Branches allow you to work on different features or fixes without affecting the main codebase.

Terminal
# List all branches
git branch

# Create a new branch
git branch feature/login

# Switch to branch
git checkout feature/login

# Create AND switch (modern shorthand)
git checkout -b feature/signup

# Delete a branch
git branch -d feature/login

Git Merge

Merging is Git's way of putting a forked history back together again.

Terminal
# Switch to target branch first
git checkout main

# Merge feature branch into main
git merge feature/login

# Abort a conflicted merge
git merge --abort

Git Remote & Push/Pull

Terminal
# Add remote
git remote add origin https://github.com/user/repo.git

# Push to remote
git push origin main

# Pull latest changes
git pull origin main

# Fetch without merging
git fetch origin
